作业一：银行家算法模拟实现
此作业实现了银行家算法，它是操作系统中用于资源分配和避免死锁的关键算法。

主要特性：
函数封装：银行家算法的核心逻辑被封装在一个独立的函数中。

灵活输入：该函数能够接收 Max（最大需求）、Need（剩余需求）、Available（可用资源）、Allocated（已分配资源）矩阵/向量，以及特定进程的 Request 向量。

决策输出：计算并判断所请求的资源是否可以安全地分配。

状态转换：如果资源被成功分配，函数将输出分配后更新的 Max、Need、Available 和 Allocated 矩阵/向量。

核心概念：
银行家算法通过模拟资源分配并检查是否仍能找到一个安全进程序列，来确保系统始终处于“安全状态”（即不会发生死锁的状态）。

作业二：使用生成器生成随机样本
此作业的重点是创建一个 Python 生成器来产生随机数据样本。

主要特性：
基于生成器：利用 Python 的 yield 关键字创建了一个高效、内存友好的生成器。

可配置样本：调用者可以指定所生成随机样本的数量和结构。

使用示例：包含演示如何使用该生成器的示例代码。

核心概念：
生成器实现了惰性求值，按需产生数据而非将所有数据一次性载入内存，这对于处理大数据集或无限序列尤为理想。

作业三：随机样本生成函数的统计操作修饰器
此作业在作业二的基础上进行了扩展，实现了一个带参修饰器，用于对生成的随机样本执行统计操作。

主要特性：
带参修饰器：一个可以接受参数（例如，'SUM'、'AVG'、'MAX'、'MIN'）以指定所需统计操作的装饰器。

灵活统计：支持 SUM、AVG、MAX 和 MIN 任意组合的统计操作。

结果输出：修饰器会修改被装饰的随机样本生成函数，使其能够输出指定的统计结果。

核心概念：
装饰器提供了一种动态地为现有函数添加功能的方式，而带参装饰器则允许通过参数配置所添加功能的具体行为，提供了更大的灵活性。

结课作业：XML 大文件解析修饰器
此最终作业将 XML 大文件解析任务重构为一个修饰器，用于修饰数据打印函数。

主要特性：
XML 解析修饰器：一个专门设计用于处理大型 XML 文件解析的修饰器。

修饰数据打印：该修饰器会包装负责打印数据的函数，将 XML 解析逻辑集成到其执行流程中。

大文件处理：重点关注高效解析大型 XML 文件，通常会避免将整个文件一次性加载到内存中。
